# Contributing

Thanks for helping improve the integration portal.

## Ways to contribute

- Fix documentation clarity issues.
- Improve non-sensitive client examples.
- Report broken links or contract mismatches.

## Ground rules

- Use placeholders in examples and requests.
- Keep examples generic and safe for public use.
- Keep changes aligned with live API behavior.

## Pull requests

- Use focused, small PRs.
- Include a clear summary and validation steps.
- Update docs when endpoint behavior changes.
